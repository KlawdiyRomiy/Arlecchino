Improved

- Snapped panel slots and floating panels now keep their content paint-ready while opening and routing between panel states.
- Settings sections and Zen chrome controls are reorganized for a cleaner settings flow.
- Local preview and project-window restore paths are more consistent when switching projects or reopening detached project windows.

Fixed

- HTML preview surfaces now stay attached to the correct project context after project switches.
- Detached project windows restore their saved state more reliably.
- Editor layout keeps the bottom edge aligned with the main workspace.

Compatibility

- Arlecchino advances to the `0.2.23-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Dependencies were refreshed for the validated release package set; TypeScript 7 and xterm 6 remain held for current peer/runtime compatibility.
- Wails runtime updates to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.

Security

- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
