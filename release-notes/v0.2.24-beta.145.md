Improved

- macOS release packages now bundle the pinned ONNX Runtime dylib, so ARLE runtime setup no longer depends on a user-installed or Homebrew runtime.
- ARLE ONNX loading and project-window restore work are deferred out of the startup path to keep launch and early UI motion lighter.
- Language detection can now hydrate lazily from generated bindings, reducing eager editor/runtime work.

Fixed

- Access-member autocomplete is more stable while typing through chained selectors.
- Frequent typing paths avoid extra heat in editor and completion state updates.

Compatibility

- Arlecchino advances to the `0.2.24-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Dependencies were refreshed for the validated release package set; `lucide-react` and `vite` were updated, while TypeScript 7 and xterm 6 remain held for current policy and peer compatibility.
- Wails runtime remains pinned to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.
- ONNX Runtime remains locked to `1.26.0` for `github.com/yalue/onnxruntime_go v1.31.0`; the universal macOS preflight validates both `arm64` and `x86_64` runtime coverage before release.

Security

- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
- Packaged ONNX Runtime archives are pinned and SHA256-verified before bundling into the release app.
