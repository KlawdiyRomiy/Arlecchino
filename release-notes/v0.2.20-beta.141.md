Improved

- Recent projects on the welcome screen can now be indexed before opening a workspace.
- Opening a pre-indexed recent project can reuse the prepared index instead of starting from an empty indexing state.
- Welcome screen context actions now cover open, new project, clone, refresh recent projects, and clear recent projects.
- Recent project entries now support copy path, reveal in file manager, remove from recent, and index/retry index actions.

Fixed

- Welcome screen recent project loading now reports failures through app notifications instead of only logging them.
- Recent project indexing status is normalized across the Wails bridge before it reaches the frontend.

Compatibility

- Arlecchino advances to the `0.2.20-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains on `v3.0.0-alpha2.113`; this is upstream dependency status, not Arlecchino product status.

Security

- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Developer ID signing and notarization will be added soon, with no date committed yet.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
