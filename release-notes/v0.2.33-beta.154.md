New

- Added the host-governed AI agent runtime with managed plugin and MCP capabilities, approval-aware tool routing, and isolated read-only or patch-artifact subagents.
- Added an interactive project dependency graph.

Improved

- AI chat continuations and follow-up steering now stay routed to the active conversation.
- Diagnostics, project filesystem updates, Git refreshes, and indexing now batch or back off work under load to keep the IDE responsive.
- Reduced interactive shell motion pressure and refined the floating shell chrome.
- Cleaned up native macOS menu entries.

Compatibility

- Arlecchino advances to the `0.2.33-beta` version line for this build.
- The normal updater is tokenless and reads the beta manifest from GitHub Releases. Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset.
- Dependencies were refreshed with stable Radix, Lucide, and RoaringBitmap updates. TypeScript 7 remains excluded, and xterm 6 remains held because the maintained canvas addon still requires xterm 5.
- Wails runtime remains pinned to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.
- ONNX Runtime remains locked to `1.26.0` for `github.com/yalue/onnxruntime_go v1.31.0`; the universal macOS preflight validates both `arm64` and `x86_64` runtime coverage before release.

Release security

- Update manifests require SHA256 and Ed25519 verification before an update can be staged.
- macOS beta artifacts use the local certificate identity for owner/tester update stability. They are not Apple Developer ID signed or notarized, so Gatekeeper can still require a manual override.
