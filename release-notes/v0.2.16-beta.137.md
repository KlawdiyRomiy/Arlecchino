Improved
- Settings now expose installed system fonts and keep IDE font choices aligned across editor, terminal, shell, tabs, and navigation surfaces.
- Theme changes now transition more smoothly, and dependency tree colors stay aligned with the active theme.
- Git history diff navigation is easier to use when moving between changed files and revisions.
- Dialog and modal surfaces avoid heavy overlay blur filters for steadier rendering.

Fixed
- Dropdown menus and definition chooser flows now handle Escape more consistently and return focus to the right surface.
- Native macOS window controls stay visible and stable during window dragging.
- Definition chooser keyboard handling is more predictable inside the editor.

Security
- Auto-update rollback protection is stricter before a staged update can replace the installed app.
- Renderer, AI, filesystem, terminal, browser preview, and release packaging paths include additional validation and safer boundary handling.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.

Compatibility
- Arlecchino advances to the `0.2.16-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains on `v3.0.0-alpha2.103`; this is upstream dependency status, not Arlecchino product status.
