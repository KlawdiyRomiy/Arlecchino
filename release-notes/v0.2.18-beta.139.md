Improved
- Project switching now hands runtime focus and editor surfaces over more smoothly when moving between project sessions.
- Quick Look preview controls have a tighter, clearer surface for opening and inspecting files.
- Dependency tree interactions now cooperate better with native macOS window controls.

Fixed
- The Wails 3 development launcher now resolves its toolchain path more reliably.
- Native macOS traffic-light controls now stay occluded behind the dependency tree when that surface is active.
- Project switch transitions now reduce stale editor and code panel state during runtime handoff.

Compatibility
- Arlecchino advances to the `0.2.18-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains on `v3.0.0-alpha2.107`; this is upstream dependency status, not Arlecchino product status.

Security
- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
