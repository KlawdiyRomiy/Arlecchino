Fixed

- Terminal sessions now close with their project, avoiding stale terminal state after a project is removed.
- Terminal startup is restored reliably after switching projects.
- Arlecchino's local MCP server now attaches consistently for discovered supported agent configurations.

Improved

- Terminal search now uses clearer match navigation and result status.
- Welcome-screen window controls are aligned with the surrounding shell.

Compatibility

- Arlecchino advances to the `0.2.30-beta` version line for this build.
- The normal updater is tokenless and reads the beta manifest from GitHub Releases. Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset.
- Dependencies were refreshed with Vite `8.1.5`. TypeScript 7 remains excluded, and xterm 6 remains held because the maintained canvas addon still requires xterm 5.
- Wails runtime remains pinned to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.
- ONNX Runtime remains locked to `1.26.0` for `github.com/yalue/onnxruntime_go v1.31.0`; the universal macOS preflight validates both `arm64` and `x86_64` runtime coverage before release.

Release security

- Update manifests require SHA256 and Ed25519 verification before an update can be staged.
- macOS beta artifacts use the local certificate identity for owner/tester update stability. They are not Apple Developer ID signed or notarized, so Gatekeeper can still require a manual override.
