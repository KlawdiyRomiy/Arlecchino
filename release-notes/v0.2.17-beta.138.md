Improved
- Modal and dialog surfaces now use a more consistent shell frame without dimming the workspace behind them.
- AI entry points can now stay hidden behind the relevant setting across command, menu, top bar, and dispatcher surfaces.
- Project explorer state now reloads more reliably after switching projects.

Fixed
- Project switch handling now refreshes explorer data instead of leaving stale file tree state behind.
- Shell menu routing and native bridge handling are more consistent with the updated hidden-AI setting.

Compatibility
- Arlecchino advances to the `0.2.17-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime is updated to `v3.0.0-alpha2.107`; this is upstream dependency status, not Arlecchino product status.

Security
- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
