Improved
- Floating panel resizing and panel motion now do less paint-heavy work during interaction.
- Frontend boot and project activation are hardened so startup and project transitions recover more cleanly.
- Shell panels, file explorer actions, and context menus keep callback state more stable across rerenders.
- LSP and diagnostics status refreshes now do less repeated work while still keeping user-visible state current.
- AI provider and runtime descriptors are cached to reduce repeated descriptor construction during agent setup.
- Indexer completion and cache behavior is more defensive around dependency and completion edge cases.

Fixed
- Diagnostic point markers and diagnostic tooltip chrome are refined for cleaner editor rendering.

Changed
- Dependency sync and AI runtime helper code paths were simplified.
- Go toolchain and static analysis configuration are now recorded in the repository.

Compatibility
- Arlecchino advances to the `0.2.10-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.98`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
