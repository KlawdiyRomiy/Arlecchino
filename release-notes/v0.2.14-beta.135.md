Improved
- LSP background work now has tighter resource bounds so language tooling puts less sustained pressure on the desktop app.
- Editor loading surfaces, file explorer transitions, and project switch surfaces now keep steadier state during active workspace changes.
- Definition quick look and Laravel definition routing are more useful when several candidate locations are available.
- Floating panels, native macOS window controls, and cursor pass-through behave more smoothly during project switches and panel interactions.

Fixed
- Diagnostics full-scan results now stay scoped to the active project more reliably.
- Project file watching now emits deleted project entries more consistently.
- Local Arlecchino metadata is hidden from workspace file listings.
- Terminal close shortcuts route through the terminal lifecycle more reliably.

Compatibility
- Arlecchino advances to the `0.2.14-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains on `v3.0.0-alpha2.103`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
