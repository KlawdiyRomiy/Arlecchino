Improved
- Editor panes now support three-way split layouts for denser side-by-side work.
- Diagnostics scanning can be started manually and cancelled, reducing background scan pressure when the workspace is active.
- Background shell status is surfaced more directly in the app chrome and diagnostics flow.
- Runtime activation pressure tracing is available across key backend bridges for performance investigation.
- Editor, chat composer, floating panel, and git status rendering now do less avoidable repaint work.

Fixed
- Native macOS window controls are corrected and can be toggled through settings without breaking window state.
- Editor and panel surfaces avoid several stale rendering paths during project and split changes.

Compatibility
- Arlecchino advances to the `0.2.11-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.98`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
