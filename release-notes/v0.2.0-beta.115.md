Improved
- Arlecchino remains on the beta release track with a focused IDE polish and release-readiness update.
- CodeMirror workflow handling is smoother, including fast-scroll rendering behavior and editor interaction extensions.
- AI chat context continuity now has a visible compact popover for inspecting and managing remembered context state.
- Custom theme examples are now documented for beta users who want to tune the IDE surface.
- Explorer scroll fog and macOS app icon override behavior received additional polish.

Fixed
- Agent patch approval handling is more defensive before applying generated code changes.
- Context continuity compaction has stronger safeguards for stale or oversized chat context.
- Packaged smoke coverage now includes runtime asset probing so release artifacts can verify bundled model/tokenizer resources more directly.

Packaging
- Local macOS signing identity support is available for release packaging flows when explicitly configured.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
