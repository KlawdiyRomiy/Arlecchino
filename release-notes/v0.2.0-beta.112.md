Changed
- Arlecchino is now beta. Product channels, packaged build defaults, update notifications, and release packaging now use the beta track.
- Existing alpha updater clients can move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Improved
- Release packaging now publishes the primary beta update manifest as `arlecchino-beta-update-manifest.json`.
- The same release can also publish the compatibility manifest `arlecchino-update-manifest.json` for already-installed alpha clients.
- GitHub release publishing now requires explicit tag, version, build, channel, notes file, a clean tracked tree, `HEAD == origin/main`, and a remote tag that points at `HEAD`.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
