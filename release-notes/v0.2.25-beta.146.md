Improved

- Public README, feature map, and third-party notices now describe Arlecchino as a desktop project workbench with current macOS beta availability and planned Windows/Linux targets.
- AI Chat surfaces get clearer runtime visibility, tighter Git/message presentation, and smoother picker behavior after runs.
- Floating panel readability and navigation motion are refined for denser IDE surfaces.
- Dependency Tree language coverage is expanded for more project-navigation contexts.

Fixed

- ONNX Runtime archive caching now self-recovers from invalid cached downloads before release or local runtime setup.
- Git panel refresh work is reduced to avoid extra background pressure.
- AI Chat command routing keeps project scope more stable.
- Patch artifact summaries are clearer when reviewing AI-generated changes.

Compatibility

- Arlecchino advances to the `0.2.25-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Dependencies were refreshed for the validated release package set; `eslint` and `prettier` were updated, while TypeScript 7 and xterm 6 remain held for current policy and peer compatibility.
- Wails runtime remains pinned to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.
- ONNX Runtime remains locked to `1.26.0` for `github.com/yalue/onnxruntime_go v1.31.0`; the universal macOS preflight validates both `arm64` and `x86_64` runtime coverage before release.

Security

- Private GitHub release manifests and update artifacts continue to use authenticated release asset access while the repository remains private.
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
- Packaged ONNX Runtime archives are pinned, SHA256-verified, and checked for valid archive contents before bundling into the release app.
