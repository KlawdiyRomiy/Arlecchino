Improved
- Arlecchino remains on the beta release track with focused Explorer and editor workflow updates.
- Explorer now supports entry history and multi-select interactions for project files.
- Find in File search chrome has been polished for the CodeMirror editor surface.

Fixed
- Dirty editor disk-change notifications are quieter and no longer show the redundant toast in that flow.
- Project entry history is persisted through the backend so recent Explorer navigation can survive normal session changes.

Compatibility
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
