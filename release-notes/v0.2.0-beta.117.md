Improved
- Arlecchino remains on the beta release track with focused editor and macOS chrome polish.
- CodeMirror no longer shows the inline lint diagnostics UI in editor surfaces where diagnostics should stay out of the editing flow.
- Code panel, diff viewer, and Quick Look editor surfaces keep the simplified diagnostics behavior aligned with the main editor.

Fixed
- macOS traffic-light controls are now more reliable after native window-control repositioning.
- Native traffic-light hit handling now forwards clicks to the standard close, minimize, and zoom buttons when custom chrome would otherwise intercept them.

Compatibility
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
