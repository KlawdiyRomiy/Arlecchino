Improved

- Project opening and detached code-panel routing preserve active editor context more reliably.
- Editor typing, floating-panel motion, and ARLE loading behavior have a lighter hot path for smoother interaction.
- AI Chat legacy controls and fallback behavior are aligned more closely with the current backend contract.

Fixed

- Startup avoids project-session bridge errors before a project is ready.
- Provider runtime shutdown is coordinated more safely, and the welcome screen no longer asks for quit confirmation unnecessarily.

Security

- Code formatting remains inside the renderer boundary, and workspace Git/file operations have tighter project-boundary checks.
- MCP, terminal, keychain, and transcript runtime inputs have additional bounds and validation.

Compatibility

- Arlecchino advances to the `0.2.26-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Dependencies were refreshed before release: stable Go and frontend updates were applied, while TypeScript 7 and xterm 6 remain held for current compatibility policy.
- Wails runtime remains pinned to `v3.0.0-alpha2.117`; this is upstream dependency status, not Arlecchino product status.
- ONNX Runtime remains locked to `1.26.0` for `github.com/yalue/onnxruntime_go v1.31.0`; the universal macOS preflight validates both `arm64` and `x86_64` runtime coverage before release.

Release delivery

- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests require SHA256 and Ed25519 verification before an update can be staged.
- Packaged ONNX Runtime archives are pinned, SHA256-verified, and checked for valid archive contents before bundling into the release app.
