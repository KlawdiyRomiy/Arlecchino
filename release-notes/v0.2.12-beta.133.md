Improved
- Interactive surfaces, popovers, dialogs, and floating panels now do less avoidable motion and repaint work during active use.
- LSP server processes now run with lower scheduling impact so language tooling puts less pressure on the desktop app.

Changed
- Regression coverage was expanded across motion deferrals, desktop editor interactions, diagnostics and indexing, access completion, dependency sync, and shared test helpers.

Compatibility
- Arlecchino advances to the `0.2.12-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.98`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
