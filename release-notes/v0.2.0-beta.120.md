Improved
- Arlecchino remains on the beta release track with focused autocomplete, LSP, and frontend health updates.
- CodeMirror completion handling now lives in a dedicated provider hook shared by the code panel surface.
- Autocomplete import handling is hardened with explicit import capability metadata and backend-backed completion resolution.

Fixed
- LSP installer registry and settings flows are more robust, with updated frontend bindings for the expanded backend contract.
- React Doctor score issues were addressed across editor, preview, AI activity, sidebar, and layout surfaces.
- Autocomplete stub providers and instant fallback completions were removed so completion behavior is driven by real backend/indexer paths.

Compatibility
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
