Improved
- Arlecchino moves to the `0.2.1-beta` version line while staying on the beta release track.
- GitHub releases are now prepared as regular latest releases, with beta status carried by the app version, update channel, and release notes.
- Update version comparison now handles semantic versions more accurately, including stable versus prerelease versions, numeric prerelease parts, and ignored build metadata.

Fixed
- macOS update code identity checks are hardened so verified update candidates preserve the expected app identity before replacement.
- Explorer filename labels no longer duplicate in the file tree.
- Git gutter deleted markers are clipped correctly in the editor surface.

Compatibility
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime is now pinned to `v3.0.0-alpha.96`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
