Improved
- Editor tabs now support split tab groups and drag workflows for file and split organization.
- Explorer files can now be dragged into editor split targets.
- Dependency tree coverage now spans more language ecosystems.
- Dialog primary button theming is centralized for more consistent project dialogs.
- Frontend dependencies were refreshed.

Fixed
- Access-member autocomplete sessions now preserve fallback truth and LSP-backed completion behavior more reliably.
- Dependency sync runtime refresh and policy/preflight state now update more consistently.
- Git marker refresh now falls back correctly when marker state changes.
- Split editor tab underline rendering and operator ligature lint ranges are corrected.

Compatibility
- Arlecchino advances to the `0.2.4-beta` version line for this build.
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts use local certificate identity stability for owner/tester updates when configured, but they are still not Apple Developer ID signed or notarized.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
