Improved
- Arlecchino remains on the beta release track with a focused macOS stability update.
- macOS open intents now route more defensively through the packaged app and window handoff paths.
- Arlecchino agent state reporting now exposes a more consistent IDE state snapshot for terminal and MCP consumers.

Fixed
- Packaged macOS builds now keep app icon and Info.plist version metadata aligned across dev, build, and Wails v3 packaging paths.
- File drops and open-file intents now preserve safer path handling before opening editor surfaces.

Compatibility
- Existing alpha updater clients can still move to this beta through the legacy alpha-channel manifest asset in the same GitHub release.
- Wails runtime remains pinned to `v3.0.0-alpha.95`; this is upstream dependency status, not Arlecchino product status.

Security
- macOS beta artifacts remain unsigned/not notarized unless Developer ID signing and notarization are explicitly configured.
- Update manifests still require SHA256 and Ed25519 verification before an update can be staged.
